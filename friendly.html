Friendly Functions

Josh Rehman, created 27 Jan 2019 last updated 20 June 2022

A friendly function is one that tells you how to call it.

  1. If you call it with nothing, it returns a pattern describing how to call it.
  2. If you call it with an object that matches the pattern, then, great!
  3. If you call it with an object that only partially matches the pattern, it will return an object containing a combination of matched values and unmatched patterns.

Important Note: Currently we do NOT return successfully matched values, and instead just return the parts that fail.

How can you tell the difference between a successful function invocation and an unsuccessful one? Well, this is an important part of the convention: return something other than an object! The most general (and useful) convention is to return an array. If you want to return a single object, return [{...}]
