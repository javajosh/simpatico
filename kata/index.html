Simple html files exercising one aspect of the browser. Serve as both exercise and reference. See kata.md for a full explanation.

  • javascript asynchrony techniques callbacks, promises, async await, rxjs, oh my.
  • site proximity coherence a classic webapp problem, with a new twist.
  • Markdown (and showdownjs)
  • mvp css (what is currently active)
  • time (browser time measurement, limits, etc)
  • websockets demo with simple chat.
  • d3 fun with d3

There are also some server-side (node) experiments:

  • zlib.js and zlib-server.js for simple node code.

Ideas for future kata
